Description

A profile of PROV designed for the representation of workflows.

Abstract

This is the specification document of the ProvWF Profile which is a profile of PROV-O, the Provenance Ontology designed for the representation of workflows. PROV-O, or the general data model it implements PROV, is a general-purpose provenance representation system whereas this profile implements specialised workflow-related elements.

This document specifies the requirements of data that must be met in order for that data to claim conformance to the ProvWF Profile.

This specification document is not to be used for testing conformance of RDF resources to this profile. That role belongs to the validation resource within this profile:

  • SHACL Validator

For the list of all resources within this profile, see the profile definition:

  • https://w3id.org/profile/provwf

1. Introduction

The ProvWF Profile defines specialised forms of the prov:Activity class, which are temporal events, specifically for describing workflows. These specialisations are:

  1. provwf:Workflow - "A Workflow is a PROV Activity that contains one or more Blocks. Workflows represent a complete system task that will likely be composed of multiple steps that should be represented by Blocks."
  2. provwf:Block - "A Block is a PROV Activity that is a part of a Workflow. Blocks are discrete tasks that use inputs and produce outputs."

There are a few properties required for use with Workflow & Block, such as provwf:hadBlock, to ensure correct correct association between instances of them.

These definitions and the complete Semantic Web ProvWF model are contained within the ProvWF Ontology and individual rules -requirements - for their use are listed below.

2. Elements & Requirements

Where model objects are indicated in code and no namespace is indicated, presume the provwf namespace is to be used.

2.1. Each Workflow & Block instance must link to at least one input which is understood to be a prov:Entity, and which it must indicate the property prov:used or a semantically equivalent construct

NOTE: Semantic equivalence here would be a construct that, through inference, can lead to the assertion of prov:used. For example, specialised forms of prov:used

2.2. Each Workflow & Block instance must link to at least one output which is understood to be a prov:Entity, and which it must indicate the property prov:generate or a semantically equivalent construct

2.3. Each Workflow & Block instance must link to the time it started with the property prov:startedAtTime and the time it ended with the property prov:endedAtTime which must indicate allowed Semantic Web date/time properties

2.3.a Allowed Semantic Web date/time properties are xsd:date, xsd:dateTime, xsd:dateTimeStamp, time:Interval

2.4. Each Workflow instance must indicate at least one Block instance within it using the property hadBlock

2.5. Each Block instance must be indicated as being within one and only one Workflow instance with the Workflow indicating the association with the via the hadBlock property

NOTE: Req. 2.4 & 2.5 are expected to be satisfied by simply ensuring that all Blcok instnaces are within Workflow instances
